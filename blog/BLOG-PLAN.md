## ✍️ 写作指南

### 文章结构

```markdown
# 标题

## 引言 (1-2 段)
- 为什么这个话题重要
- 读者将学到什么

## 背景/问题描述
- 相关背景
- 痛点/挑战

## 核心内容
### 1. 关键点 1
- 解释
- 示例

### 2. 关键点 2
- 解释
- 示例

### 3. 关键点 3
- 解释
- 示例

## 实践示例
- 完整代码示例
- 逐步讲解

## 常见问题
- Q1 + A1
- Q2 + A2

## 总结
- 回顾要点
- 行动建议

## 相关阅读
- 链接到相关文章
```

---

### 写作原则

#### ✅ 要做的事

1. **实用优先**
   - 每个观点都要有实际例子
   - 提供可运行的代码
   - 避免空洞理论

2. **结构清晰**
   - 用小标题分隔内容
   - 使用列表和表格
   - 重要内容加粗

3. **代码质量**
   - 代码要有注释
   - 使用实际场景的例子
   - 标注注意事项

4. **面向读者**
   - 假设读者有基础但不熟悉 Hermes
   - 解释专业术语
   - 提供进阶阅读链接

#### ❌ 避免的事

1. **太长不读**
   - 单篇文章控制在 2000-3000 字
   - 复杂主题分多篇写

2. **代码堆砌**
   - 不要只贴代码不解释
   - 大段代码要分段讲解

3. **缺乏重点**
   - 一篇文章解决一个问题
   - 不要试图涵盖所有内容

4. **过时信息**
   - 标注文章的 Hermes 版本
   - 定期更新旧文章

## 🚀 发布流程

### 1. 写草稿

```bash
# 在 drafts/ 目录创建草稿
cd blog/drafts
cp ../templates/post-template.md my-article.md
```

### 2. 编辑和审阅

- 自我审查（检查拼写、语法、代码）
- 技术审查（验证代码能否运行）
- 同行审查（请其他开发者阅读反馈）

### 3. 移动到 posts/

```bash
# 重命名为带序号的格式
mv my-article.md 002-my-article.md
```

### 4. 构建博客

```bash
python scripts/build-blog.py
```
